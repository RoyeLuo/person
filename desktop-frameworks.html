✨ WPF 框架 (Windows Presentation Foundation) 推荐

WPF 是 Microsoft 于 2006 年推出的 UI 框架，使用 XAML 声明式语言定义界面，配合 C# 代码后端，实现界面与逻辑的完全分离。

核心理念：使用 XAML 实现 UI 与逻辑的分离，引入数据绑定、样式、模板等现代概念，大幅提升开发效率和可维护性。

核心技术

技术 说明 优势
XAML 声明式 UI 标记语言 设计器支持、MVVM 模式
数据绑定 UI 与数据自动同步 减少样板代码
样式/模板 统一视觉风格 高度可定制
向量图形 D2D 硬件加速渲染 高清屏适配、动画流畅

XAML 示例

<Window x:Class="WpfApp.MainWindow" xmlns="http://schemas.microsoft.com/winfx/2006/xaml" Title="WPF 应用"> <StackPanel> <TextBox Text="{Binding UserName, UpdateSourceTrigger=PropertyChanged}"/> <Button Content="提交" Command="{Binding SubmitCommand}"/> </StackPanel> </Window>

MVVM 架构

  • Model - 数据模型、业务逻辑、数据访问
  • View - XAML 界面，完全被动，不知道 ViewModel 细节
  • ViewModel - 数据绑定源，实现 INotifyPropertyChanged，命令模式
// ViewModel 示例 public class MainViewModel : INotifyPropertyChanged { private string _userName; public string UserName { get => _userName; set { _userName = value; OnPropertyChanged(); } } public ICommand SubmitCommand { get; } public event PropertyChangedEventHandler PropertyChanged; protected void OnPropertyChanged([CallerMemberName] string name = null) => PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(name)); }

✓ 优势

  • 声明式 UI，开发效率高
  • 强大的数据绑定机制
  • 硬件加速，渲染性能优秀
  • 样式和模板高度可定制
  • MVVM 架构，易于测试和维护
  • 丰富的动画支持

✗ 劣势

  • 学习曲线较陡
  • Windows Only
  • 初学者容易滥用数据绑定
  • 需要 .NET 环境

🪟 WinForm 框架 传统

Windows Forms 是 Microsoft 2002 年推出的 GUI 框架，基于 .NET 的事件驱动开发模式，使用 C# 或 VB.NET。

技术特点

  • 拖拽式开发 - Visual Studio 设计器快速构建
  • 事件驱动 - 基于委托的事件处理
  • 丰富控件 - 内置 Windows 风格控件
  • RAD 开发 - 快速应用开发理念

代码示例

// WinForm 事件驱动模式 public partial class MainForm : Form { public MainForm() { InitializeComponent(); this.btnClick.Click += BtnClick_Click; } private void BtnClick_Click(object sender, EventArgs e) { MessageBox.Show("按钮被点击!"); } }

✓ 优势

  • 上手简单，开发效率高
  • VS 设计器支持完善
  • 学习资源丰富
  • 适合快速原型

✗ 劣势

  • 界面定制能力有限
  • 无 MVVM/MVP 支持
  • 代码与 UI 混杂
  • 无硬件加速

🔧 MFC 框架 (Microsoft Foundation Classes) 传统

MFC 是 Microsoft 1992 年推出的 C++ 类库，对 Win32 API 的面向对象封装，支持文档/视图架构。

技术特点

  • C++封装 - 对 Win32 API 的 OOP 封装
  • 消息映射 - DECLARE_MESSAGE_MAP 宏驱动
  • 文档/视图 - 内置 MDI/SDI 架构
  • 高性能 - 编译型语言，接近底层

消息映射示例

// MFC 消息映射 class CMainFrame : public CFrameWnd { protected: afx_msg void OnBnClickedButton(); DECLARE_MESSAGE_MAP() }; BEGIN_MESSAGE_MAP(CMainFrame, CFrameWnd) ON_BN_CLICKED(IDC_MYBUTTON, &CMainFrame::OnBnClickedButton) END_MESSAGE_MAP()

✓ 优势

  • 性能优秀
  • 可直接操作 Win32 API
  • 单文件 exe 部署
  • 成熟稳定

✗ 劣势

  • 学习曲线陡峭
  • 代码复杂，宏多
  • Windows Only
  • 现代 UI 支持有限

🎯 框架选择建议

选择 WPF 当：

  • 开发企业级桌面应用
  • 需要现代化 UI 和流畅动画
  • 重视代码可维护性和可测试性
  • 团队熟悉 MVVM 架构
  • 需要复杂的数据绑定场景

选择 WinForm 当：

  • 快速开发内部工具
  • 项目周期短，需快速交付
  • 维护现有 WinForm 系统
  • 团队 .NET 基础较弱

选择 MFC 当：

  • 对性能有极高要求
  • 需要直接操作 Win32 API
  • 维护大型 MFC 遗留系统
  • 需要单文件 exe 部署
总结：WPF 是 Windows 桌面开发的现代主流选择， 结合 MVVM 模式能够构建可维护、可测试的企业级应用。 新项目建议优先选择 WPF。
