创建型模式

5种

单例模式 Singleton

创建型
保证一个类只有一个实例，并提供一个全局访问点。
应用场景：数据库连接池、配置文件管理、日志记录器、全局唯一ID生成器 
public class Singleton {
    private static volatile Singleton instance;
    
    private Singleton() {}
    
    public static Singleton getInstance() {
        if (instance == null) {
            synchronized(Singleton.class) {
                if (instance == null) {
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}

工厂方法 Factory Method

创建型
定义创建对象的接口，让子类决定实例化哪个类。
应用场景：日志记录器（文件/数据库/云存储）、支付网关（支付宝/微信/银联） 
interface LoggerFactory {
    Logger createLogger();
}
class FileLoggerFactory implements LoggerFactory {
    public Logger createLogger() {
        return new FileLogger();
    }
}

抽象工厂 Abstract Factory

创建型
创建一系列相关对象的家族，而无需指定具体类。
应用场景：UI主题（Windows/Mac/Linux组件）、数据库访问（MySQL/Oracle/PostgreSQL） 
interface AbstractFactory {
    Button createButton();
    TextBox createTextBox();
}
class WindowsFactory implements AbstractFactory {
    public Button createButton() { return new WindowsButton(); }
    public TextBox createTextBox() { return new WindowsTextBox(); }
}

建造者 Builder

创建型
将一个复杂对象的构建与表示分离，允许分步构建。
应用场景：SQL查询构建器、HTTP请求构建、复杂配置文件创建、DTO转换 
User.Builder builder = new User.Builder();
User user = builder
    .name("张三")
    .age(25)
    .email("zhang@example.com")
    .phone("13800138000")
    .build();

原型模式 Prototype

创建型
通过复制现有对象来创建新对象，无需知道具体类型。
应用场景：对象克隆、缓存对象恢复、避免重复创建复杂对象 
class Resume implements Cloneable {
    private String name;
    public Resume clone() {
        try {
            return (Resume) super.clone();
        } catch (CloneNotSupportedException e) {
            return null;
        }
    }
}

结构型模式

4种

适配器 Adapter

结构型
将一个类的接口转换成客户期望的另一个接口。
应用场景：第三方SDK集成、接口版本兼容、遗留系统对接 
interface Target {
    void request();
}
class Adaptee {
    public void specificRequest() {}
}
class Adapter implements Target {
    private Adaptee adaptee;
    public void request() {
        adaptee.specificRequest();
    }
}

装饰器 Decorator

结构型
动态给对象添加额外职责，比继承更灵活。
应用场景：IO流包装、日志增强、缓存层、权限校验 
interface DataSource {
    void write(String data);
    String read();
}
class EncryptionDecorator extends DataSourceDecorator {
    public void write(String data) {
        super.write(encrypt(data));
    }
}

代理模式 Proxy

结构型
为其他对象提供一种代理以控制对这个对象的访问。
应用场景：延迟加载、远程代理、安全代理、缓存代理、事务管理 
interface Image {
    void display();
}
class ProxyImage implements Image {
    private RealImage realImage;
    public void display() {
        if (realImage == null) {
            realImage = new RealImage(this.fileName);
        }
        realImage.display();
    }
}

外观模式 Facade

结构型
为复杂的子系统提供一个统一的高层接口。
应用场景：微服务网关、系统集成、简化第三方库调用 
class ComputerFacade {
    private CPU cpu;
    private Memory memory;
    private Disk disk;
    
    public void start() {
        cpu.start();
        memory.load();
        disk.read();
    }
}

行为型模式

3种

策略模式 Strategy

行为型
定义一系列算法，将每个算法封装起来，使它们可以互换。
应用场景：支付方式选择、排序算法切换、促销计算、压缩策略 
interface SortStrategy {
    int[] sort(int[] array);
}
class QuickSort implements SortStrategy { ... }
class MergeSort implements SortStrategy { ... }
class Sorter {
    private SortStrategy strategy;
    public void setStrategy(SortStrategy s) { this.strategy = s; }
}

观察者模式 Observer

行为型
定义对象间的一对多依赖关系，当对象变化时通知所有依赖者。
应用场景：事件监听、消息推送、GUI交互、数据绑定、订单状态变更通知 
interface Observer {
    void update(String message);
}
class Subject {
    private List observers = new ArrayList<>();
    public void attach(Observer o) { observers.add(o); }
    public void notify() {
        observers.forEach(o -> o.update(message));
    }
}

模板方法 Template Method

行为型
定义算法骨架，将某些步骤延迟到子类中实现。
应用场景：框架初始化流程、数据处理 pipeline、单元测试基类 
abstract class DataMiner {
    // 模板方法
    public final void mine(String path) {
        String file = openFile(path);
        String data = extractData(file);
        String parsed = parseData(data);
        sendReport(parsed);
    }
    abstract String openFile(String path);
}
