Docker基础

Docker是一个开源的容器化平台，让应用打包、分发、运行变得简单高效。

核心概念

Image 镜像

只读模板，包含应用和依赖

可层层构建版本化

Container 容器

镜像的运行实例，相互隔离

轻量快速可写层

Volume 卷

持久化数据，容器间共享

数据持久挂载

Registry 仓库

存储和分发镜像

Docker Hub私有仓库

常用命令

# 镜像操作
docker images                          # 列出本地镜像
docker pull nginx:latest               # 拉取镜像
docker rmi nginx:latest                # 删除镜像
docker build -t myapp:1.0 .            # 构建镜像
docker tag myapp:1.0 registry.com/myapp # 打标签
docker push registry.com/myapp         # 推送镜像

# 容器操作
docker ps -a                           # 列出所有容器
docker run -d -p 8080:80 --name web nginx  # 运行容器
docker start/stop/restart web          # 启停容器
docker exec -it web /bin/bash          # 进入容器
docker logs -f web                      # 查看日志
docker rm -f web                        # 删除容器

# 清理
docker system prune -f                 # 清理无用资源

run 参数详解

docker run [OPTIONS] IMAGE [COMMAND]

# 核心参数
-d                    # 后台运行
-p 8080:80            # 端口映射（主机:容器）
-v /host/path:/container/path  # 目录挂载
-e KEY=VALUE          # 环境变量
--name myapp          # 容器名称
--network mynet       # 加入网络
--restart unless-stopped  # 自动重启策略
--memory 512m         # 内存限制
--cpus 1.0            # CPU限制
--rm                  # 停止后自动删除
-w /app               # 工作目录

Dockerfile最佳实践

# 多阶段构建示例
# 阶段1：构建
FROM maven:3.8-openjdk-17 AS builder
WORKDIR /app
COPY pom.xml .
RUN mvn dependency:go-offline    # 缓存依赖层
COPY src ./src
RUN mvn package -DskipTests

# 阶段2：运行（精简镜像）
FROM eclipse-temurin:17-jre-alpine
WORKDIR /app
COPY --from=builder /app/target/myapp.jar ./app.jar

# 安全配置
RUN addgroup -S appgroup && adduser -S appuser -G appgroup
USER appuser

# 健康检查
HEALTHCHECK --interval=30s --timeout=3s --retries=3 \
    CMD curl -f http://localhost:8080/health || exit 1

# 使用非root用户
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]
优化建议：
  • 合理利用层缓存：频繁变化的内容放最后
  • .dockerignore：排除不需要的文件
  • 最小化基础镜像：alpine、distroless
  • 合并RUN指令：减少镜像层数
  • 多阶段构建：分离构建和运行环境

Docker Compose

定义和运行多容器Docker应用的工具。

# docker-compose.yml
version: "3.8"

services:
  # Web应用
  webapp:
    build: .
    ports:
      - "8080:8080"
    environment:
      - SPRING_PROFILES_ACTIVE=prod
      - DB_HOST=postgres
    depends_on:
      postgres:
        condition: service_healthy
    networks:
      - backend
    deploy:
      resources:
        limits:
          memory: 512M

  # 数据库
  postgres:
    image: postgres:15-alpine
    volumes:
      - postgres_data:/var/lib/postgresql/data
    environment:
      POSTGRES_DB: myapp
      POSTGRES_USER: admin
      POSTGRES_PASSWORD: ${DB_PASSWORD}
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U admin"]
      interval: 10s
      timeout: 5s
      retries: 5
    networks:
      - backend

  # Redis缓存
  redis:
    image: redis:7-alpine
    command: redis-server --appendonly yes
    volumes:
      - redis_data:/data
    networks:
      - backend

  # Nginx反向代理
  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf
    depends_on:
      - webapp
    networks:
      - backend

volumes:
  postgres_data:
  redis_data:

networks:
  backend:
# 常用命令
docker-compose up -d                  # 启动所有服务
docker-compose down -v                # 停止并删除
docker-compose ps                      # 查看状态
docker-compose logs -f webapp          # 查看日志
docker-compose exec webapp bash        # 进入容器
docker-compose restart webapp          # 重启服务
docker-compose build --no-cache        # 重新构建
docker-compose scale webapp=3          # 扩容

Kubernetes基础

K8s是容器编排平台，提供自动化部署、扩缩容、服务发现等功能。

核心概念

Pod

最小调度单元，包含一个或多个容器

共享网络共享存储

Deployment

管理Pod副本，支持滚动更新

声明式回滚

Service

Pod的负载均衡和网络抽象

ClusterIPNodePortLoadBalancer

Ingress

HTTP/HTTPS路由，外部访问入口

域名SSL

ConfigMap/Secret

配置数据和敏感信息

环境变量挂载

PersistentVolume

持久化存储抽象

持久化

Deployment示例

apiVersion: apps/v1
kind: Deployment
metadata:
  name: myapp
spec:
  replicas: 3
  selector:
    matchLabels:
      app: myapp
  template:
    metadata:
      labels:
        app: myapp
    spec:
      containers:
      - name: myapp
        image: myapp:1.0
        ports:
        - containerPort: 8080
        resources:
          limits:
            memory: "256Mi"
            cpu: "500m"
          requests:
            memory: "128Mi"
            cpu: "250m"
        livenessProbe:
          httpGet:
            path: /health
            port: 8080
          initialDelaySeconds: 30
          periodSeconds: 10
        readinessProbe:
          httpGet:
            path: /ready
            port: 8080
          initialDelaySeconds: 5
          periodSeconds: 5

常用命令

kubectl get pods                         # 查看Pod
kubectl get svc                           # 查看Service
kubectl get all                           # 查看所有资源
kubectl describe pod myapp-xxx           # 查看详情
kubectl logs -f pod/myapp-xxx             # 查看日志
kubectl exec -it myapp-xxx -- /bin/bash   # 进入容器
kubectl apply -f deployment.yaml          # 部署
kubectl scale deployment myapp --replicas=5  # 扩缩容
kubectl rollout status deployment/myapp   # 查看滚动更新
kubectl rollout undo deployment/myapp      # 回滚

CI/CD流水线

持续集成/持续部署自动化软件交付流程。

典型流水线阶段

1. Code Checkout

拉取代码、切换分支

2. Build

编译构建（mvn/gradle/npm）

3. Test

单元测试、集成测试

4. Security Scan

代码扫描、依赖漏洞检测

5. Docker Build

构建镜像、打标签

6. Push

推送到镜像仓库

7. Deploy

部署到测试/生产环境

8. Verify

健康检查、烟雾测试

Jenkinsfile示例

pipeline {
    agent { label 'docker' }
    
    environment {
        REGISTRY = 'registry.com'
        IMAGE_NAME = "myapp/${BRANCH_NAME}"
    }
    
    stages {
        stage('Checkout') {
            steps {
                checkout scm
            }
        }
        
        stage('Build') {
            steps {
                sh 'mvn clean package -DskipTests'
            }
        }
        
        stage('Test') {
            steps {
                sh 'mvn test'
            }
            post {
                always {
                    junit 'target/surefire-reports/*.xml'
                }
            }
        }
        
        stage('Docker Build & Push') {
            when { branch 'main' }
            steps {
                sh """
                    docker build -t ${IMAGE_NAME}:${BUILD_NUMBER} .
                    docker push ${IMAGE_NAME}:${BUILD_NUMBER}
                """
            }
        }
        
        stage('Deploy to K8s') {
            when { branch 'main' }
            steps {
                sh "kubectl set image deployment/myapp myapp=${IMAGE_NAME}:${BUILD_NUMBER}"
            }
        }
    }
}

# Fluentd 配置示例
<source>
  @type tail
  path /var/log/myapp/*.log
  pos_file /var/log/fluentd/myapp.log.pos
  tag myapp.*
  <parse>
    @type json
  </parse>
</source>

<match myapp.*>
  @type elasticsearch
  host elasticsearch
  port 9200
  logstash_format true
  logstash_prefix myapp
</match>
可观测性三支柱：
  • Metrics - 量化指标（Prometheus）
  • Logs - 事件日志（ELK）
  • Traces - 请求链路（Jaeger/Zipkin）
